# 模型化A股系统定义

本文档把原策略文档中的概念定义成系统字段和计算逻辑。

## 三因子

| 因子 | 系统名称 | 数据源 | 系统含义 |
| --- | --- | --- | --- |
| 总量分母 | `liquidity_denominator` | 中美利率、美元指数、人民币汇率、A股成交额、融资余额、指数波动率 | 决定市场风险偏好和估值分母 |
| 总量分子 | `macro_numerator` | PMI、CPI/PPI、工业企业利润、社融、指数盈利、行业盈利趋势 | 决定盈利总量和总量 beta |
| 结构分子 | `structure_numerator` | 主题池、板块强弱、成交占比、港美映射、商品价格、产业事件、订单/产能/资本开支、人工叙事 | 决定结构性超额收益 |

系统必须提供独立的“三因子”工作台，不能只在驾驶舱中展示总分。该页面至少展示每个因子的定义、当前评分、乐观/中性/悲观判断、判断摘要、权重、置信度、数据字段、正负信号、指标明细和证据链，用于解释中期基线为什么成立或失效。

三因子不能静态求和后直接给结论。系统必须动态评估三者的相对强弱和方向，并输出：

- 市场主导力量：例如结构分子占优、总量分子转弱、总量分母压制、总量共振。
- 因子强弱排序：展示总量分母、总量分子、结构分子的相对位置。
- 中期基线方向：例如“分母中性，结构分子占优”时，基线方向应偏结构性做多，战略上不应看空市场。
- 战略指导：把主导格局翻译成仓位和象限选择。

结构分子不能只靠行情涨跌判断。系统按四层证据链判断：

1. 主题确认：人工或事件录入确认赛道叙事，例如产业资本开支、供给约束、新能源渗透率。
2. 行情验证：检查板块相对强弱、成交占比、趋势稳定性、扩散宽度和熵值。
3. 外部锚验证：检查港股、美股、商品价格或产业链上游价格是否共振。
4. 微观证据确认：检查订单、产能、资本开支、供需缺口、公司指引是否支撑叙事。

只有叙事、资金和产业证据同时改善，结构分子才应判为乐观；只有情绪上涨但缺少产业或外部锚验证时，只能归入短期故事或三象限交易对象。

## 中期基线

中期基线是慢变量趋势锚，由三因子和结构分子共同决定。系统输出：主叙事、方向、多空状态、核心锚资产、核心板块、置信度、失效条件。更新频率为日度检查、周度确认；除非因子明显变盘，否则不频繁切换。

系统必须把中期基线作为交易引擎的“锚点”，输出红绿灯和过滤器：

- 红绿灯：由三因子战略判断和中期基线状态共同决定，控制是否允许围绕主线做多。
- 过滤器：只优先处理符合中期叙事、锚资产和核心板块的对象；不符合中期叙事的方向只能作为短期试错。
- 交易模式：趋势交易、相对收益、方向选择和仓位锚定，不追求极致择时。

## 短期基线

短期基线是围绕中期基线波动的故事、情绪和波动率抓手。系统输出：当前故事、传播强度、情绪温度、波动状态、参与板块、与中期基线连接度。短期故事可以独立交易，但必须能从中期基线偷逻辑；完全脱离中期基线的故事只能低仓位、短周期处理。

系统必须把短期基线作为“情绪与波动率抓手”，并输出三大纪律：

- 偶要断：短期故事按情绪和波动率交易，不用中期基本面解释每次高频波动。
- 丝要连：故事根基必须从中期基线偷逻辑，否则只能低仓位、短周期。
- 情绪监控：用连接度、情绪温度、波动状态、成交拥挤度、筹码结构或信息熵监控风险。

中短基线必须并行输出共振状态。当短期故事与中期趋势方向一致、连接度高、情绪温度支持时，系统输出“中短共振”；共振越强，允许的手部交易积极度越高。若短期故事脱离中期基线，则不能升级为中期仓位，只能按三象限短打。

## 四区间

| 区间 | 系统 ID | 状态 | 动作 |
| --- | --- | --- | --- |
| 区间一 | `interval_1_discount` | 情绪过度悲观，核心品种低熵，价格低于中期基线 | 增持一象限核心 |
| 区间二 | `interval_2_disorder` | 贴水修复后资金分散，市场选主线 | 选好车，跟随强势集群 |
| 区间三 | `interval_3_premium` | 情绪过热，后排扩散 | 守核心、攻弹性，控制三象限风险 |
| 区间四 | `interval_4_reversion` | 升水结束，波动放大，高熵退潮 | 防守，减三象限，回到低熵核心 |

四区间是短期基线交易行为与中期基线互动后呈现出的择时模型。系统必须输出完整策略表，包括区间、核心状态、三因子与基线特征、策略基调和交易哲学。

区间循环动力：

1. 区间一：讲故事。情绪过度悲观，短期基线开始从中期基线偷逻辑。
2. 区间二：信故事。市场开始相信故事，通过图形和集群行为选择主线。
3. 区间三：共识扩散。核心扩散到后排，情绪惯性主导。
4. 区间四：情绪回归现实。故事溢价消退，资金重新回到中期基线附近。

短期基线负责推动“讲故事-信故事-市场共识扩散-情绪回归现实”，中期基线负责评判当前价格相对基线是贴水还是升水。

## 四象限

四象限是给定策略区间下，对品种进行资金属性与定价逻辑分类的择股工具。横轴是资金属性，从游资情绪到机构共识；纵轴是定价逻辑，从叙事驱动到基本面驱动。

| 象限 | 系统 ID | 坐标特征 | 资金属性 | 定价逻辑 | 代表品种 | 在四区间的角色 |
| --- | --- | --- | --- | --- | --- | --- |
| 一象限 | `quadrant_1_core` | 机构共识 + 基本面驱动 | 公募、保险、外资等大资金 | 数据驱动、中长期景气度 | 光模块、铜铝、黄金 | 核心屁股：在区间一、四增持；在区间三守护 |
| 二象限 | `quadrant_2_trading` | 游资情绪 + 基本面驱动 | 游资、量化、部分灵活机构 | 数据驱动但弹性更大 | 国产算力、光纤光缆、储能材料 | 手部主场：高抛低吸，博取中期基线延伸的情绪溢价 |
| 三象限 | `quadrant_3_hunt` | 游资情绪 + 叙事驱动 | 纯游资、散户、高风险偏好量化 | 纯情绪和远期预期 | 商业航天、AI 应用早期、油气 | 对手盘/猎场：在区间三用于套利；区间一、四需规避 |
| 四象限 | `quadrant_4_transition` | 机构共识 + 叙事驱动 | 游资抱团后获得机构认可 | 叙事向基本面过渡 | PCB 上游、从三象限成功突围的品种 | 过渡区：从三象限向一二象限迁移的桥梁 |

系统必须展示品种归类证据，而不是只展示象限名。每个品种至少给出机构共识分、基本面证据分、情绪资金分、叙事强度分、熵值、象限评分、当前动作和归类理由。

核心应用：

- 区间一：重点增持一象限，吃贴水，规避三象限高波叙事。
- 区间二：所有象限“选好车”，尊重市场选择出的主线和核心品种。
- 区间三：重一守三攻，一象限守核心，二/三象限做手部加速。
- 区间四：右缩左扩，缩三象限高熵品种，扩一象限低熵核心。

## 决策矩阵

- 区间一 + 一象限：增持核心低熵品种。
- 区间二 + 全象限：寻找市场自然选择出的强势主线。
- 区间三 + 一/二/三象限：一象限守仓，二象限高抛低吸，三象限快进快出。
- 区间四 + 三象限：减仓或退出，资金回流一象限和部分四象限。

## 数据源后台

后台必须能查看和配置数据源，而不是只隐藏在 ETL 脚本中。

| 数据源 | 系统 ID | 写入表 | 用途 |
| --- | --- | --- | --- |
| AKShare A股行情 | `akshare_a_spot` | `market_prices`, `market_breadth` | 市场宽度、情绪、热度、个股基础行情 |
| AKShare 行业与概念板块 | `akshare_boards` | `themes`, `narrative_events`, `quadrant_items` | 自动识别结构分子、短期故事和象限候选 |
| AKShare 宏观慢变量 | `akshare_macro` | `macro_observations` | 自动派生总量分子，缺失时低置信度中性处理 |
| yfinance 港美与商品锚 | `yfinance_anchors` | `anchor_assets` | 中期基线外部锚 |
| 模型状态自动派生 | `model_derivation` | `factor_states`, `baseline_states`, `quadrant_items` | 根据原始数据自动生成三因子、中短基线和象限候选 |
| 宏观慢变量人工补强 | `manual_macro` | `macro_observations`, `factor_states` | 自动宏观采集失败或口径不合适时人工覆盖 |
| 结构叙事与产业事件人工补强 | `manual_narratives` | `themes`, `narrative_events`, `baseline_states`, `quadrant_items` | 产业订单、产能、公司指引和人工确认 |
| 收盘后采集任务 | `github_actions_etl` | `data_collection_runs` 等 | Python ETL 编排与日志记录 |
| Cloudflare 健康检查 | `cloudflare_health` | `data_sources`, `data_collection_runs` | API 和配置状态检查 |

Cloudflare 后台负责配置、状态查看和 API；GitHub Actions 负责运行 Python 采集。每次采集必须写入 `data_collection_runs`，包括数据源、状态、开始/结束时间、写入行数、错误消息和 GitHub Actions 日志链接。

## 自动派生层

系统不能依赖固定样例或要求用户手工填写模型状态。`scripts/collect_market_data.py` 在 GitHub Actions 中完成以下链路：

1. `akshare_a_spot` 写入全市场行情和市场宽度，生成情绪、成交热度、波动、离散度和贴水/升水基础分。
2. `akshare_boards` 读取行业/概念板块强弱，自动识别当日强势主题，并写入主题、叙事事件和象限候选。
3. `akshare_macro` 尝试读取 PMI、CPI、PPI、社融等宏观慢变量；如果接口缺失或失败，总量分子保持中性低置信度，不编造方向。
4. `yfinance_anchors` 写入商品、美元、美债、汇率等外部锚，参与总量分母和中期基线确认。
5. `model_derivation` 根据上述原始数据写入 `factor_states`、`baseline_states`、`quadrant_items`。

自动派生规则：

- 总量分母：由成交热度、市场情绪、波动压力和美元/美债/汇率外部压力合成。
- 总量分子：优先由宏观慢变量合成；缺失时中性低置信度。
- 结构分子：由行业/概念板块相对强弱、扩散宽度、成交热度和外部锚合成。
- 中期基线：由三因子和前列强势主题生成主叙事、核心板块、锚资产和失效条件。
- 短期基线：由高热主题、情绪、波动率以及与中期主题的重合度生成。
- 四象限：由自动识别主题的机构共识分、基本面证据分、情绪资金分、叙事强度分、趋势分、熵值和流动性分分类。

人工录入不是主路径，只用于补强自动数据无法验证的产业证据，例如订单、产能、资本开支、供需缺口和公司指引。

数据源页必须可编辑，并至少支持修改启停、调度、新鲜度、目标表、用途、责任方、配置 JSON 和备注。配置 JSON 中必须保留采集来源字段，例如 `script`、`package`、`function`、`tickers`、`maxRows`，让使用者能直接判断数据来自 AKShare、yfinance、人工录入还是 GitHub Actions 编排。
